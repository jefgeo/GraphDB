A simple graph solution.
* The code creates Nodes and a set of Nodes with metadata to describe the entities.
* Relationships connect the Nodes.

The initial version here was designed to show relationships between individuals and their strengths.
* Nodes are either an individual or a strength
* Relationships connect an individual to their strengths.
* Since the relationship is bi-directional, it is easy to query an individual's strengths
  or the individuals who have a given strength

Docstrings are provided for each class and method.

main.py and detailed_report.py show usage
